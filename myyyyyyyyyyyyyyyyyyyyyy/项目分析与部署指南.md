# Vue Vben Admin 项目完整分析与部署指南

## 一、项目概述

| 属性 | 值 |
|------|-----|
| 项目名称 | vben-admin-monorepo |
| 版本 | 5.5.9 |
| 许可证 | MIT |
| 项目类型 | Monorepo (单仓多包) |
| 官方文档 | https://doc.vben.pro/ |
| 在线演示 | https://vben.pro/ (账号: vben/123456) |

**Vue Vben Admin** 是一个免费开源的中后台管理系统模板，采用最新的 Vue 3、Vite、TypeScript 等主流技术开发，提供开箱即用的中后台前端解决方案。

## 五、Windows 部署步骤

### 步骤 1: 验证环境 (已满足)

打开 PowerShell 或命令提示符，验证环境：

```powershell
# 检查 Node.js 版本 (需要 >=20.19.0)
node --version
# 输出: v22.14.0 ✅

# 检查 pnpm 版本 (需要 >=10.0.0)
pnpm --version
# 输出: 10.28.2 ✅
```

**你的环境已经满足要求，无需额外安装！**

### 步骤 2: 安装项目依赖

```powershell
# 进入项目目录
cd "G:\双创\ShuangC\frontend\vue-vben-admin-main"

# 安装依赖 (首次安装可能需要几分钟)
pnpm install
```

> **注意**: 如果遇到网络问题，可以先设置镜像源：
> ```powershell
> pnpm config set registry https://registry.npmmirror.com
> ```

### 步骤 3: 启动开发服务器

项目提供了多个版本，你可以选择其中一个启动：

```powershell
# 方式1: 启动所有应用 (会同时启动多个)
pnpm dev

# 方式2: 只启动 Ant Design Vue 版本 (推荐)
pnpm dev:antd

# 方式3: 只启动 Element Plus 版本
pnpm dev:ele

# 方式4: 只启动 Naive UI 版本
pnpm dev:naive

# 方式5: 只启动 TDesign 版本
pnpm dev:tdesign

# 方式6: 启动文档站点
pnpm dev:docs

# 方式7: 启动演练场
pnpm dev:play
```

启动成功后，终端会显示类似如下信息：
```
  VITE v7.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: http://192.168.x.x:5173/
```

在浏览器中打开 `http://localhost:5173/` 即可访问。

### 步骤 4: 构建生产版本

```powershell
# 构建所有应用
pnpm build

# 或只构建特定版本
pnpm build:antd      # Ant Design Vue 版本
pnpm build:ele       # Element Plus 版本
pnpm build:naive     # Naive UI 版本
pnpm build:tdesign   # TDesign 版本
pnpm build:docs      # 文档站点
```

构建产物位于各应用的 `dist/` 目录下。

### 步骤 5: 预览生产构建

```powershell
pnpm preview
```

## 八、常见问题解决

### Q1: 安装依赖时报错

```powershell
# 清理缓存后重新安装
pnpm store prune
pnpm install
```

### Q2: 端口被占用

修改 `apps/web-antd/.env.development` 中的端口配置，或使用其他终端关闭占用端口的进程。

### Q3: 内存不足

项目构建时已设置 `NODE_OPTIONS=--max-old-space-size=8192`，确保系统有足够内存 (建议 8GB+)。

### Q4: 如何切换 UI 框架

直接使用对应的命令即可：
- Ant Design: `pnpm dev:antd`
- Element Plus: `pnpm dev:ele`
- Naive UI: `pnpm dev:naive`
- TDesign: `pnpm dev:tdesign`

### Q5: 如何清理并重新安装

```powershell
pnpm reinstall
```
